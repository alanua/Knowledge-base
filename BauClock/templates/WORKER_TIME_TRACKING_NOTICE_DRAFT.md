# BauClock — Worker Time Tracking Notice Draft

## Status

Draft worker-facing time tracking notice for onboarding and registration flow.
This is a product template, not final legal advice.

---

## 1. Purpose

Your company uses BauClock to record working time and related site-attendance events.
The system may record:
- work start
- break start
- break end
- work end

---

## 2. How the workflow works

Typical workflow:
- you scan a site QR or use the assigned check-in path
- the system records a time event
- repeated actions may move through break and end-of-day flow
- the system can show daily/weekly/monthly summaries derived from these events

---

## 3. Why the company uses this system

The system is used to support:
- working-time recording
- site attendance documentation
- summary calculation
- payment-support workflows where applicable
- review of disputes or corrections where needed

---

## 4. Manual corrections

If a manager/owner/admin manually changes a time-related record, the system should mark that correction and log who made it and why.

---

## 5. Your role

You are expected to:
- use the check-in/check-out flow correctly
- report problems promptly
- review your own visible summaries where applicable
- use confirmation/dispute actions if payment-support workflow requires them

---

## 6. Acknowledgement note

For real product use, the worker should acknowledge receiving this notice.
This template must be reviewed and finalized before client use.
