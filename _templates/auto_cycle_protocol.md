# Auto Cycle Protocol

## Goal
Use a controlled improvement loop for all projects.

## Loop
observe -> detect -> propose -> validate -> approve -> apply -> monitor

## Rules
- do not change project goals
- do not change architecture without approval
- do not change production without review
- do not store secrets in the knowledge base
- update the knowledge base in every cycle

## Inputs
- tasks.md
- handoff.md
- decisions.md
- bug reports
- review results
- runtime observations

## Outputs
- next proposed task
- risk note
- validation plan
- knowledge-base update

## Stop conditions
- architecture conflict
- missing context
- unclear validation
- change too large for one step

## Required record
1. observation
2. detected issue or opportunity
3. proposed action
4. validation method
5. approval state
6. result
7. follow-up
